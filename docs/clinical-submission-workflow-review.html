Clinical Submission Flow Review

临床送检菜单与流程优化评审稿

本稿不新增“送检工作站”大页面，而是把侧边栏菜单按真实岗位与连续操作重新收敛： 建单和登记合并，固定和转运合并，病理接收与追踪异常保留独立。

结论

从 6 个可见节点收敛为 4 个可见菜单，旧路径隐藏 redirect 兼容。

4 申请与登记 / 固定与转运 / 病理接收 / 追踪与异常
0 不新增独立送检工作站，不再让用户保存后跳转再查询。

当前现状

真实操作链

当前链路分散为：申请管理 -> 创建/导入弹窗 -> 标本管理 -> 二次查询申请单 -> 登记标本 -> 固定核对 -> 转运交接 -> 标本接收 -> 追踪查询。

主要断点
  • 建单与登记割裂，保存后上下文丢失，需要二次输入申请单号。
  • 固定与转运前后连续，但菜单分散，未固定标本拦截不够直观。
  • 申请单关键字段和时间链缺口影响后续质控追溯。
  • 接收异常能记录，但质量判定、原因分类和追踪闭环不够突出。

优化后流程

菜单按“同阶段、强连续、可共享上下文才合并”的原则调整为 4 个入口。

申请与登记

建单、导入、去重、登记

  • 保留申请单列表、筛选、创建和导入。
  • 成功后当前页打开登记抽屉。
  • 登记抽屉带入患者、申请单、离体时间和随附状态。
固定与转运

固定核对后再交接

  • 固定核对和转运交接以 Tab 或分区承接。
  • 转运区只展示已固定标本。
  • 从申请与登记带 applicationId 自动筛选。
病理接收

病理端质控判定

  • 按转运单聚合接收。
  • 支持接收、退回整改、拒收、部分接收。
  • 明细必须填写质控结果和必要原因码。
追踪与异常

跨角色查询与闭环

  • 按申请单号、applicationId、条码查询。
  • 展示离体、固定、转运、接收全时间链。
  • 突出异常标记、退回/拒收原因和处理状态。
旧路径 `/workflow/application-list`、`/workflow/specimen-management`、`/workflow/clinical-register`、 `/workflow/fixation-verify`、`/workflow/transport-handover`、`/workflow/specimen-receipt`、 `/workflow/tracking-query` 全部隐藏并 redirect 到新入口。

岗位泳道

医生

  • 提交临床诊断、送检部位、申请类型和医嘱来源。
  • 确认外部申请号和疑似重复申请。

护士 / 送检登记

  • 补齐离体时间、申请单随附状态、容器与部位信息。
  • 完成标本登记、标签打印和标签补打。

转运

  • 只接收已完成固定的标本清单。
  • 创建转运单、打印、交接并记录操作人和终端。

病理接收

  • 核对容器、标签、固定、数量和污染渗漏。
  • 按质量结果执行接收、退回整改或拒收。

必填与校验点

菜单 必填信息 关键校验
申请与登记 申请日期、送检日期、来源医院、送检科室、送检医生、临床诊断、送检部位、离体时间、申请单随附状态。 外部申请号重复强拦截；同患者同日同部位同类型提示人工确认；登记抽屉不再要求二次查询。
固定与转运 开始固定时间、固定完成时间、固定液、操作人、终端、备注、交接人。 未固定标本不进入可转运清单；完成固定后才能创建转运单。
病理接收 容器数量、质控结果、问题代码、退回/拒收原因、接收人。 正常接收必须质控合格；质控不合格必须选问题代码；退回/拒收必须填结构化原因。
追踪与异常 申请单号、applicationId、条码、异常标记、拒收/退回原因、关键时间链。 前序页面异常跳转自动带查询条件；部分接收、拒收、退回必须可追溯。

异常分支

重复申请

  • 外部申请号重复返回 BLOCK 时禁止保存。
  • 同患者同日同部位同类型返回 CONFIRM 时必须人工确认。

未固定

  • 固定状态未完成的标本不出现在转运可选清单。
  • 从申请与登记跳转时只自动筛选，不绕过固定核对。

标签不符

  • 标签缺失或申请单与标本不一致时不能正常接收。
  • 必须选择问题代码并走退回整改或拒收。

部分接收 / 拒收退回

  • 部分接收自动产生异常标记。
  • 追踪与异常页展示原因码、备注、责任节点和后续处理状态。

系统改造清单

前端

  • 侧边栏收敛为 4 个可见菜单。
  • 申请与登记、固定与转运内部承接上下文。
  • 旧路径隐藏 redirect 并保留 query。

接口

  • 复用申请单扩展字段和重复预警接口。
  • 接收请求继续传质控结果与问题代码。
  • 不新增后端接口。

数据

  • 菜单种子/迁移同步新名称与新路径。
  • 旧菜单记录隐藏兼容，权限码继续复用。
  • 申请单随附状态只做枚举状态。

追踪

  • 贯穿离体、固定、转运、接收节点。
  • 异常原因、质量问题与部分接收可查询。
  • 前序页面支持带条件跳转追踪。

流程原则参考： CAP Specimen Handling、 Johns Hopkins Surgical Pathology、 UT Health Specimen Requirements。

2026-05 评审补充

登记要素

  • 标本登记核心要素升级为标本名称、标本部位、标本数量、容器名称、容器数量。
  • 容器名称采用“字典建议 + 可手输”，v1 仅保存文本快照，不引入单独容器编码。
  • 条码仍保持选填，但填写后继续受唯一性约束。

页面入口

  • 申请与登记中的登记页签已恢复，登记入口改为轻量弹窗。
  • 登记弹窗不再保留申请单查询区域，沿用工作台上下文直达。
  • 新增标本行继续放在行操作体系，不回退为独立大按钮。

管理与核验

  • 标本管理列表与详情已补齐容器名称、容器数/标本数展示。
  • 标签打印状态和操作列补齐补打标签、异常处理、开始核验、完成核验入口。
  • “已核验”筛选语义调整为固定完成，不再依赖标签打印成功状态。

后续节点透传

  • 接收页默认继承登记侧容器数量，并展示容器名称上下文。
  • 接收环节仍允许修正容器数量，作为实际接收结果。
  • 工作流参考字典新增病理标本容器名称分类，支持前后端统一维护。
