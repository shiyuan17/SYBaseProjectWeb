# T-004 技术医嘱工作站后端动作契约补齐

## Goal

补齐特检、IHC、细胞学、液基细胞学工作站所需的后端动作契约，支撑前端从“壳子页”升级为真实工作站。

## Inputs

- 来源：`docs/reviews/full-process-remediation-plan-2026-07-01.html`
- 审计依据：`docs/reviews/full-process-audit-2026-07-01.html`
- 对照对象：常规医嘱工作站现有动作链

## Outputs

- 四类工作站缺失的接口、命令对象、返回模型和状态变化规则
- 每个动作对应的权限校验、状态校验、落库和日志要求
- 前端接线所需的稳定动作契约

## Constraints

- 设计要对齐常规医嘱工作站的交互模式，不额外发散新范式
- 每个关键动作都必须定义后端状态变化和失败语义
- 不接受“只有页面按钮、没有真实业务动作”的半接入状态

## Acceptance Criteria

- 四类工作站的关键动作都有明确后端接口与返回契约
- 每个动作都补齐权限、状态、异常和日志处理
- 输出契约足以支撑 `T-005` 前端接线
