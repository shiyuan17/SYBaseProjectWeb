# T-011 归档借阅与白片库存并发安全加固

## Goal

为归档借阅和白片库存相关操作补齐并发安全机制，避免重复借出、库存覆盖和状态脏写。

## Inputs

- 来源：`docs/reviews/full-process-remediation-plan-2026-07-01.html`
- 依赖任务：`T-010`
- 目标范围：材料借阅、借阅归还、白片借记、白片归还

## Outputs

- 乐观锁版本、条件更新或悲观锁方案
- 409 冲突返回规范
- 并发冲突相关测试

## Constraints

- 事务不等于并发安全，不能只保留“先查再改”模式
- 需要与审批流引入后的借阅状态机兼容
- 冲突提示必须可被前端和人工操作理解

## Acceptance Criteria

- 同一材料并发借阅只能成功一次
- 白片库存并发借出/归还不会出现负数或覆盖
- 并发冲突会返回稳定且可识别的业务错误
