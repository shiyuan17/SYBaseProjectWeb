# T-014 主状态与分发状态展示规范统一

## Goal

统一前端对主状态、分发状态和异常状态的展示规范，避免把不同生命周期混成一个标签。

## Inputs

- 来源：`docs/reviews/full-process-remediation-plan-2026-07-01.html`
- 依赖任务：`T-008`、`T-010`、`T-012`、`T-013`

## Outputs

- 报告主状态与报告分发状态的分开展示规范
- 标本、技术、报告、借阅的状态文案和颜色约定
- 相关前端格式化工具与页面调整

## Constraints

- 不能再用单个标签概括“已签发/已发布”和“已打印/已发放”
- 需要和新的状态模型、tracking 数据结构一致
- 以用户可理解和可追责为目标，而不是只做 UI 美化

## Acceptance Criteria

- 页面能清楚区分主状态与分发状态
- 标本、报告、借阅关键状态的文案与颜色规则一致
- 业务人员无需查看源码即可理解当前对象所处阶段
