# T-006 技术任务池权限拆分与服务校验

## Goal

拆分技术任务池写操作权限，并在后端控制器和服务层同时建立正确的权限边界。

## Inputs

- 来源：`docs/reviews/full-process-remediation-plan-2026-07-01.html`
- 审计依据：技术任务相关权限全部复用 `TECHNICAL_TASK_QUERY`

## Outputs

- 新的技术任务权限码：查询、分派、领取、释放、改优先级、改备注
- 控制器动作级权限注解调整
- 服务层的岗位、归属、越权写操作防护

## Constraints

- 不能只做注解替换，必须补服务层二次校验
- 需要同步考虑权限 seed、角色授权和前端能力判断
- 不改变已完成任务不可写的既有约束

## Acceptance Criteria

- 只有查询权限的账号无法执行任一写操作
- 技术任务写操作具备明确动作级权限码
- 服务层能拒绝不符合岗位/归属规则的越权操作
