# T-013 技术登记接口收敛

## Goal

收敛技术登记 `detail` 与 `workspace` 等重复或重叠接口，明确主用契约和消费者边界。

## Inputs

- 来源：`docs/reviews/full-process-remediation-plan-2026-07-01.html`
- 审计依据：前端存在定义但运行时未明显使用的技术登记 detail 接口

## Outputs

- 技术登记主用接口契约决策
- 对重复接口的保留、下线或兼容策略
- 调整后的前端调用链和测试

## Constraints

- 收敛时不能破坏现有技术登记工作台核心使用路径
- 若保留兼容接口，需要明确其只读/兼容定位
- 必须减少重复字段映射和职责重叠

## Acceptance Criteria

- 技术登记存在一条明确的主用契约
- 前端运行时代码不再依赖无意义的重复接口定义
- 对保留的兼容接口有明确范围说明和测试覆盖
