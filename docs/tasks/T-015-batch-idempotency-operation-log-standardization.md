# T-015 批量动作幂等与操作日志标准化

## Goal

统一批量动作的回执、幂等和操作日志正文结构，提升排障与复盘能力。

## Inputs

- 来源：`docs/reviews/full-process-remediation-plan-2026-07-01.html`
- 依赖任务：`T-008`、`T-011`
- 目标范围：优先覆盖报告分发、借阅、库存等高风险批量动作

## Outputs

- 标准批量回执结构
- 重复提交防护和幂等策略
- 更可读的操作日志正文：业务对象、动作、结果、关键摘要

## Constraints

- 优先收敛已经在整改范围内的高风险批量动作，不做全仓无差别重构
- 不允许只记录 HTTP method / uri，而缺少业务动作摘要
- 幂等策略要与状态机和并发控制方案兼容

## Acceptance Criteria

- 重复提交高风险批量动作时行为可预测、结果可解释
- 操作日志能直接说明“谁对什么对象做了什么、结果如何”
- 批量接口成功/失败项结构一致，便于前端和审计系统消费
