02. 测试准备
测试前先统一环境、角色、测试数据和设备条件，减少中途阻塞。

环境准备

  • 前端环境：部署当前可访问的 web-ele 测试版本。
  • 后端环境：对接当前 bl-center 测试环境，接口路由可正常访问。
  • 建议浏览器：现代 Chromium 内核浏览器；若有国产浏览器要求，需追加专项记录。
  • 联调方式：优先使用真实测试环境，不用本地 Mock 替代主闭环验收。

账号准备

  • 优先准备后端种子测试用户，避免现场手工造角色导致权限口径不一致。
  • 如需重建账号，先由 M1 管理员按本手册完成用户创建和角色分配。
  • 建议在测试前确认每个账号可正常登录，并能看到对应菜单目录。

数据准备

  • 常规病理测试病例 1 组，用于完整 M2 → M3 → M4 主链路。
  • 第三方导入测试数据 1 组，用于 M2 临床申请导入路径。
  • 返工场景测试病例 1 组，用于 M3 返工补充链路。
  • 修订和会诊测试病例 1 组，用于 M4 附加闭环。

设备与现场说明

  • 标签打印、转运单打印、登录标签打印需至少验证一次成功反馈。
  • 若现场具备扫码枪，建议优先用扫码路径执行 M2 固定、接收与追踪。
  • 真机打印、扫码枪失败回退、浏览器兼容性需单独在结果记录区说明。
执行建议 先完成 M1 账号与权限准备，再按 M2 → M3 → M4 顺序走主闭环。
导入说明 HIS/EMR 导入路径依赖真实测试数据；若导入失败，应记录数据来源、外部单号和错误码。
边界提醒 本文不把 M4 医嘱执行页面当作前端正式步骤，避免把后端已具备能力误写成当前前端已支持。

04. 跨模块总流程
把 M1 视作测试前置准备，M2-M4 组成常规病理主闭环。
M1 系统管理 创建用户、分配角色、核对菜单和权限
→
M2 标本送检 申请单、登记、固定、转运、接收、追踪
→
M3 技术组 取材、脱水、包埋、切片、染色、追踪
→
M4 医生组 分配、诊断、报告、审核、签发、发布、追踪
执行顺序 推荐完整顺序：M1 账号准备 → M2 送检 → M3 技术 → M4 医生。
主闭环核心 从 M2 创建申请单开始，到 M4 报告发布结束，是本次最重要的全链路。
补充链路 M2 导入、M3 返工、M4 修订与会诊属于补充验证，不影响主闭环先走通。

05. M1 系统管理执行手册

目标是先由管理员准备后续闭环所需账号、角色、菜单和权限，并验证登录标签、导入导出与基础数据维护能力。

岗位：病理管理员 角色：ROLE_PATHOLOGY_ADMIN 用户：USER_M1_ADMIN
病理管理员 用户管理流
创建测试用户 在系统用户页新建 M2/M3/M4 所需测试账号
分配岗位角色 为各账号设置主角色与对应岗位角色
验证菜单可见性 重新登录对应账号，确认只看到应有菜单
验证标签与导出 检查登录标签、用户导入导出反馈
病理管理员 角色授权流
进入角色授权 核对目标角色的菜单树与权限树
保存授权配置 保存菜单、权限、消息主题、统计范围
回登录验证 用目标岗位账号确认入口、按钮和页面权限
病理管理员 / 用户管理流
用于准备后续模块测试账号，并验证用户工作台能力。
ROLE_PATHOLOGY_ADMIN USER_M1_ADMIN

入口页面

  • /system/users
  • 同模块可补查 /system/configs、/system/numbering-rules

前置条件

  • 管理员账号可正常登录。
  • 后端角色种子已存在。
  • 已明确本轮要用的岗位账号列表。

操作步骤

  1. 进入系统用户页，按测试计划创建缺失账号。
  2. 分别为账号分配目标岗位角色，并设置主角色。
  3. 查看登录日志和用户状态，确认账号处于启用状态。
  4. 执行一次登录标签打印，记录成功或失败反馈。
  5. 按需执行用户导出；若现场需要，执行导入模板验证。

预期结果 / 失败点

  • 预期：账号创建成功，角色保存成功，状态可见。
  • 预期：登录标签与导入导出有明确成功反馈。
  • 失败点：角色保存不生效、标签打印失败、导入模板不匹配。
病理管理员 / 角色授权流
用于核对岗位权限和菜单授权，确保后续模块测试不会因角色问题中断。
ROLE_PATHOLOGY_ADMIN USER_M1_ADMIN

入口页面

  • /system/roles

前置条件

  • 目标角色已存在，例如 ROLE_M2_CLINICAL_REGISTER。
  • 测试账号已关联对应角色。

操作步骤

  1. 依次打开 M2、M3、M4 目标岗位角色。
  2. 核对菜单授权是否包含对应工作流目录和岗位页面。
  3. 核对权限授权是否覆盖该岗位所需关键动作。
  4. 保存角色授权后，用对应测试账号回登录检查。
  5. 如页面按钮显隐异常，回看菜单和权限树是否缺项。

预期结果 / 失败点

  • 预期：菜单授权和权限授权保存成功。
  • 预期：目标岗位账号重新登录后只看到本岗位页面。
  • 失败点：页面可见但操作按钮缺失，通常是权限树未授权。

最小闭环

  • 创建 1 个岗位测试用户
  • 分配 1 个岗位角色
  • 目标账号重新登录看到正确菜单
  • 角色授权保存成功

关键异常

  • 重复用户编码或账号名
  • 角色分配后菜单未刷新
  • 登录标签打印失败
  • 导入模板与系统字段不匹配

未覆盖项

  • 真实打印机兼容性
  • 大批量导入性能
  • 更细粒度审计策略确认

ROLE_M2_CLINICAL_REGISTER USER_M2_REGISTER

入口页面

前置条件

操作步骤

  1. 创建申请单并确认当前上下文已切换到新申请单。

预期结果 / 失败点

  • 预期：申请单创建成功并能加载详情。
  • 预期：标本登记成功，标签结果可见。
  • 失败点：上下文未切换、详情数组缺省、打印反馈不明确。

预期结果 / 失败点

  • 失败点：无导入权限、外部单号重复、HIS/EMR 数据缺失。

ROLE_M2_FIXATION_VERIFY USER_M2_FIXATION

入口页面

  • 已知测试条码或可在待固定列表筛出该条记录。

  1. 在待固定列表中定位目标标本，或扫描条码填入操作区。

  4. 刷新列表，确认待固定数量减少或状态更新。

预期结果 / 失败点

  • 预期：开始和完成固定均提示成功。
  • 失败点：重复固定、空操作人、状态不可逆。

  • 预期：申请单和条码路径都能看到一致事件链。

最小闭环

  • 创建或导入申请单
  • 登记标本并完成标签补打
  • 完成固定、转运、接收
  • 申请单和条码追踪均可查询

关键异常

  • 重复导入 / 重复接收
  • 固定与接收必填项缺失
  • 打印动作失败或反馈不明确
  • 拒收、退回原因校验失败

未覆盖项

  • 真机打印链路稳定性
  • 扫码枪高频输入体验
  • 真实 HIS/EMR 批量导入策略

  3. 主持人完成会诊，并回看工作台和追踪摘要。

修订预期 / 失败点

关键异常

  • 审核驳回后重新提交流程
  • 跨角色误用：非审核人审核、非签发人签发
  • 发布后状态未关闭或版本链缺失
  • 修订和会诊摘要未回流工作台/追踪页
