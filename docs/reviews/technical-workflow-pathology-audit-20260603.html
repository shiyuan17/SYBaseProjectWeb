Pathology Workflow Audit

制片管理流程与标准病理系统对照审查报告

本报告基于当前前端仓库 SYBaseProjectWeb 与同级后端仓库 SYBaseProject 的实际代码状态形成，聚焦“制片管理流程是否已经具备标准病理系统可用性”。结论是：主链已经基本打通，但关键工作站仍存在统计口径失真、占位动作未闭环、追踪可见性不足、患者核对信息缺失等问题，距离“可审计、可质控、可规模上线”的标准病理技术流程仍有明显差距。

审查日期 2026-06-03 范围: 前端 + 后端 重点: 制片主链 / 任务池 / 追踪 结论: 可演示, 未达稳态上线
主链已存在 接收 -> 技术登记 -> 取材 -> 脱水 -> 包埋 -> 切片 -> 染色 -> 诊断移交
4 项高优先级 统计口径、工作站闭环、筛选分页一致性、错误可见性需要优先修正
建议 4 阶段推进 先统一口径与高风险节点，再补站点闭环，再做质控与归档增强

审查范围与方法

纳入范围

  • 前端模块：specimen-workflow、technical-workflow、doctor-workflow
  • 后端接口：技术登记、任务池、取材、脱水、包埋、切片、染色、诊断、报告追踪
  • 内部基线：后端规划文档 病理全流程系统里程碑规划.md 的 M3/M4 目标
  • 对照基线：通用病理系统主链、异常分支闭环、追溯与质控要求

审查口径

  • 不是只看页面是否存在，而是看前后端是否真的形成一致的数据流和可操作闭环
  • 优先关注病理现场高风险点：患者核对、玻片追踪、统计可信度、异常返工、审计留痕
  • 结论以代码现状为准，不替代真实医院 SOP、条码设备和现场角色权限联调

现状总判断

总体判断

当前系统不是“完全没有制片流程”，而是“主干流程已搭起，但关键节点还停留在半成品状态”。前端路由和工作站已经覆盖技术组全链路，后端也具备从染色完成自动移交诊断的闭环逻辑，说明系统架构方向是对的；但如果以标准病理系统要求衡量，当前仍缺少稳定的口径定义、完整的异常分支、可靠的患者与玻片核对信息，以及对错误和追踪结果的充分暴露。

1. 接收 / 技术登记

前端已有技术标本登记页，后端已有待登记、工作台、核对、完成接口。

2. 任务池 / 工位流转

任务池统一承接取材、脱水、包埋、切片、染色等任务，方向符合标准生产线设计。

3. 染色完成

后端在无剩余染色任务时可把病例推进到 DIAGNOSIS_PENDING。

4. 医生承接

前端医生工作台与报告追踪页存在，后端提供聚合视图接口，闭环已经具备骨架。

问题清单

高优先级

技术登记筛选与分页口径不一致

前端在 TechnicalSpecimenRegistrationView.vue:105-112 用 visiblePendingItems 本地过滤 applicationType，但请求 463-469 没有把这个字段传给后端，随后又直接把后端 result.total 赋给列表总数 470-471。

  • 风险：总数、分页、当前页展示结果不一致。
  • 病理影响：技师看到的待办量和实际可处理量不一致，容易导致漏处理或误判排班压力。
高优先级

染色工作站统计口径不可信

StainingWorkstationView.vue:164-186 中“当日应切总数”直接等于 total，“未出片”也直接等于 total，“已出片”却来自当前追踪结果的玻片计数。与此同时， stainingType 虽然有筛选控件 113, 430-436，但未见进入后端查询。

  • 风险：统计卡片不是同一时间范围、同一数据集、同一业务口径。
  • 病理影响：班组长和质控人员会基于错误指标做生产判断，属于管理级高风险缺陷。
高优先级

切片工作站关键异常动作仍是占位

SlicingWorkstationView.vue:85-89 明确存在 LEGACY_PLACEHOLDER_ACTIONS， 包含重打玻片、重新分片、重补台片、终止等关键动作； 605-610 的提示为“待补后端能力”， 736-740 还显示“取消完成仅保留占位”。

  • 风险：异常分支无法系统内闭环，只能依赖线下口头或纸面补救。
  • 病理影响：重切、重打、终止等本应是病理技术流程的核心受控动作，缺失后审计链会断裂。
高优先级

报告追踪错误被静默隐藏

ReportTrackingView.vue:56, 85-90 已维护 pageError，但 199-205 的错误告警组件被写成 v-if="false"。

  • 风险：追踪查询失败时，页面无法提供稳定错误反馈。
  • 病理影响：报告追踪本来就是问题排查入口，静默失败会拖慢诊断与客服协同。
中优先级

取材工作站仍有多处显性未接入

GrossingWorkstationView.vue:571 写明“暂未接入日期筛选”，602 写明“暂未接入”， 867-869 写明“暂未接入独立暂存”。

  • 风险：取材工位的检索、暂存和辅助动作不完整。
  • 病理影响：取材是上游高风险节点，任何临时动作如果不留痕，后续包埋与切片追踪都会受影响。
中优先级

染色工作站缺少患者关键信息

StainingWorkstationView.vue:123-124, 142-143 将 patientId 和 patientName 直接写成 '-'。

  • 风险：玻片出片前的人员核对维度不足。
  • 病理影响：标准病理系统通常要求患者、病理号、玻片号至少形成双重或三重核对，本页当前做不到。
中优先级

技术登记存在合成回退，可能掩盖真实数据缺口

后端 TechnicalSpecimenRegistrationService.java:551, 569 在缺少真实登记记录时会走 createSyntheticRegistration。

  • 正面作用：兼容旧数据和迁移期病例。
  • 风险：前台看起来“有登记记录”，底层却未必存在完整登记实体和操作历史。
  • 病理影响：审计、追责、数据修复和稽核时很容易误判完成度。
低优先级

登记默认术语尚未标准化

TechnicalSpecimenRegistrationService.java:35-38 存在默认值 SURGERY、 OPERATING_ROOM、 Specimen Bottle、 FORMALIN。

  • 风险：术语和院内字典不一致时，前后端展示、报表分组和培训资料会出现偏差。
  • 病理影响：不是马上阻断流程的问题，但会影响标准化交付与后续多院区复制。

前后端对齐建议

1. 统一查询口径

所有工作站的筛选项必须在请求参数、分页总数、统计卡片和导出结果中使用同一口径，避免前端本地二次过滤制造“假总数”。

2. 统一状态与异常编码

重切、重打、取消完成、终止、独立暂存、返工原因、超时原因都应有后端状态机和审计事件，而不是仅停留在按钮文案。

3. 统一追踪时间线

技术事件、诊断事件、医嘱事件、质控事件和报告版本变更要能在同一病例维度串起来，避免“技术端看一套、医生端看一套”。

4. 统一质控指标

当日应切、已切、未出片、已出片、超时、返工等指标先做定义表，再决定前后端实现；禁止页面直接用零散字段拼统计。

5. 统一患者核对模型

染色、出片、诊断接收都应展示患者姓名、病理号、标本号/蜡块号/玻片号等关键信息，必要时接扫码核对。

6. 统一术语字典

采样场景、固定液、容器、申请类型、标本类型等术语要使用院内主数据字典，不建议长期保留英文硬编码默认值。

分段执行计划建议

Phase 0: 口径冻结与风险清单确认（3-5 个工作日）

  • 冻结技术流程主数据：任务状态、异常动作、统计指标、申请类型、患者核对字段。
  • 梳理现有页面与后端接口的参数矩阵，标出哪些筛选未下推后端。
  • 输出一版“技术流程指标字典”和“异常动作编码表”。

Phase 1: 高风险闭环修正（1-2 周）

  • 修正技术登记分页与筛选不一致问题。
  • 重构染色统计口径，改为单一后端统计接口或统一聚合查询。
  • 恢复报告追踪错误可见性，细分错误场景。
  • 补足染色页患者核对字段，至少形成病理号 + 患者 + 玻片号核对。

Phase 2: 工位异常动作补齐（2-4 周）

  • 优先补切片站的重打玻片、重新分片、重补台片、取消完成、终止。
  • 补取材站的暂存、日期筛选、关键辅助动作。
  • 确保所有异常动作都有：权限、原因、事件记录、回退后的状态定义。

Phase 3: 质控与追踪增强（2-3 周）

  • 把技术质控、返工记录、超时记录纳入统一病例级时间线。
  • 对合成登记增加显式标识和稽核报表，推动旧数据清理或迁移补录。
  • 统一前后端术语字典与默认值，收口为院内主数据配置。

Phase 4: 试运行验收（1-2 周）

  • 按技师、医生、管理员三类角色做端到端演练。
  • 验证主链、异常返工、追踪查询、统计报表和审计导出。
  • 形成上线前验收表，确认哪些节点仍需人工 SOP 补位。

建议验收标准

必须达成

  • 任一筛选条件下，待办列表、分页总数、统计卡片结果一致。
  • 重切、重打、终止、取消完成等高频异常动作可在系统内闭环并留痕。
  • 染色和出片节点可展示完整核对字段。
  • 报告追踪出现失败时，用户能明确看到原因并继续排查。

建议达成

  • 合成登记病例可被筛出并单独审计。
  • 统计指标在前后端共用统一定义文档。
  • 技术事件与医生事件能以病例为单位串联展示。
  • 默认术语全部改为字典驱动或院内可配置。

备注：本报告认定“当前制片管理流程不是不可用，而是尚未达到标准病理系统稳态要求”。如果下一步只允许做最小改动，建议优先执行 Phase 1；如果目标是支撑真实科室试运行，则至少应完成 Phase 2。
