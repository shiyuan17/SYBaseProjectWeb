# 管理员岗位 SOP

## 适用范围

- 角色定位：病理系统管理员、病理科管理员、试点演练组织人
- 试点范围：`M1` 系统管理、`M5` 归档管理/试剂台账/设备台账、Phase 1.5 演练编排与验收材料留档
- 本轮目标：确保四类角色账号、权限、菜单、M5 前端页面、演练记录与问题台账全部可用

## 演练前检查

- 已完成四类岗位账号启用、密码确认和角色分配
- 已核对菜单映射，`/operation-support/archive`、`/operation-support/reagents`、`/operation-support/equipment` 可按角色访问
- 已准备联调环境、测试数据、应急通讯录、问题台账模板和验收记录
- 已明确本轮不扩展采购、收费、真实文件上传、Excel 导出和统计报表

## 主流程

1. 演练前 1 个工作日核对四类角色账号、角色菜单和关键权限，确认前端入口与后端接口均可访问。
2. 组织护士、技师、医生按主链路完成一次最小闭环演练，管理员同步记录时间点、问题和人工兜底动作。
3. 在 `M5` 页面完成归档柜、归档记录、借阅记录、试剂台账、库存预警、设备台账和保养预警的核对。
4. 对权限缺失、菜单错误、回流缺失、页面报错等问题即时建账，并指定责任人和临时兜底方案。
5. 演练结束后汇总前端命令、后端测试、现场问题、阻断项数量和复核结论。
6. 更新验收记录，确保文档、截图、命令结果和问题台账能够支撑 Phase 1.5 交付。

## 异常处理

| 场景 | 处理动作 | 验收口径 |
| --- | --- | --- |
| 页面入口丢失或跳转错误 | 先核对角色菜单与前端路由映射，再核查后端菜单配置 | 需明确是菜单配置、权限还是前端映射问题 |
| 角色可见页面但调用接口 403 | 比对角色权限码与前端按钮权限 | 同一问题需保留账号、接口、权限码三项证据 |
| 归档/借阅/预警接口返回异常 | 先确认测试数据是否完整，再定位接口日志 | 未查清前需提供人工业务兜底 |
| 演练数据脏写或状态冲突 | 停止复用该病例，改用新的测试病例继续演练 | 原病例进入问题台账并标注不可复用 |
| 测试命令失败 | 记录失败命令、错误摘要、最后通过位置 | 阻断项必须在验收记录中单独统计 |

## 权限缺失

- 表现：四类岗位页面不可见、按钮灰置、接口 403、菜单跳空白页
- 自查顺序：
  1. 核对用户与角色绑定
  2. 核对角色与菜单/权限绑定
  3. 核对前端 authority 配置与菜单映射
  4. 核对后端 `PERM_*` 权限码与控制器注解
- 临时策略：
  - 明确禁止共享账号或越权代操作
  - 使用纸质记录或管理员陪同记录完成现场流程
  - 当日内补齐权限并安排复测

## 人工兜底

- 现场流程中断：由管理员统一协调纸质单据、电话确认和班后补录
- M5 页面不可用：由管理员直接通过后端验收记录、日志和测试数据做人工确认
- 多角色同时受影响：暂停试点节奏，先恢复账号和关键权限，再重新安排演练

## 复核记录

- 复核人：项目负责人、病理科管理员或试点负责人
- 复核时点：演练前、演练后、验收结论输出前
- 复核内容：
  - 四类 SOP 是否齐全
  - 演练记录是否包含主流程、异常、兜底和联系人
  - 问题台账是否区分阻断项和一般项
  - 前后端命令结果是否已留档
- 复核留痕模板：

| 日期 | 检查项 | 结果 | 阻断项数 | 复核意见 | 复核人 |
| --- | --- | --- | --- | --- | --- |
| YYYY-MM-DD | 账号 / 权限 / 演练 / 验收 | 通过 / 不通过 | 0 | | |

## 升级联系人

- 一级：病理系统管理员
- 二级：病理科管理员 / 试点负责人
- 三级：前端联调负责人 / 后端接口负责人
- 四级：发布负责人或项目经理
