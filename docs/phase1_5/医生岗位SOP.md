# 医生岗位 SOP

## 适用范围

- 角色定位：诊断医生、审核医生、签发医生
- 试点范围：`M4` 诊断分配、工作台、报告、追踪，以及 `M5` 归档回流状态核对
- 本轮目标：验证医生在不直接维护 `M5` 台账的前提下，能在工作台和追踪页看到准确的归档、借阅回流信息，并完成诊断主闭环

## 演练前检查

- 账号具备诊断任务查询、分配/接单/开始、报告保存/提交/审核/签发/发布权限
- 工作台可查看标本、蜡块、玻片摘要与追踪链
- 已知本轮不开放采购、收费、真实文件上传、统计报表能力

## 主流程

1. 在分配或待办页面领取病例，确认病例号、申请单号、送检类型和优先级。
2. 进入诊断工作台，核对标本、蜡块、玻片、历史记录、医嘱和技术流转信息。
3. 在报告页面完成草稿保存、提交、审核、签发和发布，必要时发起会诊或修订。
4. 在工作台或追踪页核对 `applicationFormArchiveStatus`、`applicationFormArchiveLocation`、`applicationFormImageUrl` 以及蜡块/玻片的 `archiveStatus`、`archiveLocation`、`loanStatus`。
5. 若需要查阅已归档材料，先在追踪页确认位置，再联系归档管理员发起借阅；医生仅核对回流结果，不直接修改 M5 台账。
6. 发布前完成病例主结论复核，确认归档或借阅信息不会影响当前出报告结论。

## 异常处理

| 场景 | 处理动作 | 验收口径 |
| --- | --- | --- |
| 工作台无归档回流字段 | 先刷新页面并重新进入病例；仍缺失时记录病例号并升级 | 视为 M4/M5 联调缺陷，需要登记台账 |
| 蜡块/玻片状态与现场不一致 | 暂停借阅决策，联系归档管理员核对借阅记录 | 未确认前不得写入错误结论 |
| 申请单图片地址失效 | 使用纸质申请单或人工拍照件复核，并通知管理员补查 | 需记录是否影响报告出具 |
| 诊断或修订提交失败 | 保留草稿内容，截图错误信息并联系管理员 | 不允许因提交失败丢失诊断文本 |
| 医嘱执行结果未回流 | 联系技师确认是否已执行或是否存在返工 | 报告中不得假设医嘱已完成 |

## 权限缺失

- 表现：无法进入工作台、无法保存或提交流程、无法查看追踪页
- 自查顺序：
  1. 确认当前账号为医生岗位账号
  2. 核对是否误用审核岗或只读岗账号
  3. 由管理员核对 `M4` 权限与菜单映射
- 临时策略：
  - 不使用他人账号代签、代审
  - 使用纸质诊断单或会诊单记录意见，待权限恢复后补录
  - 在问题台账中登记病例号、步骤和报错截图

## 人工兜底

- 系统无法查看归档回流：改由归档管理员电话或纸质借阅单反馈材料位置
- 报告流程异常：保留草稿文本和截图，先人工签批内部意见，不对外发布错误版本
- 会诊或修订流程受阻：由科主任指定人工审批路径，并要求事后补录系统

## 复核记录

- 复核人：审核医生、签发医生或科主任授权复核人
- 复核时点：报告提交前、签发前、借阅申请前
- 复核内容：
  - 诊断结论与工作台信息一致
  - 技术链和医嘱链状态可解释
  - 归档/借阅回流字段已核对
  - 修订或会诊意见已留痕
- 复核留痕模板：

| 日期 | 病例号 | 报告阶段 | 回流核对结果 | 异常说明 | 复核人 |
| --- | --- | --- | --- | --- | --- |
| YYYY-MM-DD | | 草稿 / 审核 / 签发 | 通过 / 不通过 | | |

## 升级联系人

- 一级：当班主诊医生或审核医生
- 二级：病理科主任或诊断组负责人
- 三级：归档管理员 / 系统管理员
- 四级：Phase 1.5 前端联调负责人 / 后端接口负责人
