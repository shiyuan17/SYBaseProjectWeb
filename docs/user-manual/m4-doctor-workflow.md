# M4 诊断管理

诊断管理围绕任务分配、诊断工作站、报告管理、追踪、医嘱执行、报告修订与会诊协作展开。

流程：诊断分配 -> 诊断平台工作站 -> 病理报告 -> 报告追踪 -> 病理医嘱执行 -> 报告修订 -> 会诊管理

### 1.任务分配与诊断工作站

用于把病例分派给医生，并在统一工作站内完成诊断与报告编辑。

#### 1.1 诊断分配

菜单路径：诊断管理 -> 诊断分配

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 进入诊断分配后，先在左侧选择医生或查看当前医生负荷。
2. 再从右侧待分派列表中勾选任务，执行初步分片或签发分片。

可选操作：查询待分派任务；选择医生；执行初步分片或签发分片

预期结果：任务会按医生分派，作为后续进入诊断工作站的前置步骤。

#### 1.2 诊断平台工作站

菜单路径：诊断管理 -> 诊断平台工作站

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 在工作站中先从任务队列选择病例，再查看报告编辑区和资料区。
2. 根据实际诊断进度进行保存、初步、复核、签发或发放等操作。

可选操作：选择任务；编辑报告；保存或签发；查看医嘱、会诊与历史病理

预期结果：工作站应同时展示任务队列、报告编辑和上下文资料，支持连续诊断。

### 2.报告管理与追踪

用于统一查看报告状态、追踪病例生命周期并联动诊断环节。

#### 2.1 病理报告

菜单路径：诊断管理 -> 病理报告

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 进入病理报告列表后，按病例、病理号或状态进行检索。
2. 根据当前状态执行驳回、发布、打印、发放或回收等操作。

可选操作：按病例查询；驳回或发布报告；打印、发放或回收正式版本

预期结果：报告列表可作为诊断结果统一管理与版本处理入口。

![病理报告列表：用于查询病例报告并执行驳回、发布与版本处理。](images/m4-doctor-workflow/m4-report-overview.png)

#### 2.2 报告追踪

菜单路径：诊断管理 -> 报告追踪

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 进入报告追踪后，按病例号或病理号查询指定病例。
2. 查看病例时间线、对象流转和关键节点，必要时跳转回工作站或报告列表。

可选操作：按病例查询；查看时间线；跳转工作站或医嘱页面

预期结果：报告追踪可只读回看病例全局生命周期，便于定位流转环节。

### 3.医嘱、修订与会诊协作

用于处理诊断侧的医嘱执行、报告修订和多医生协作会诊。

#### 3.1 病理医嘱执行

菜单路径：诊断管理 -> 病理医嘱执行

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 打开医嘱执行页面后，按病理号、状态或类型检索目标医嘱。
2. 根据医嘱流程执行确认、打印玻片、出片或取消等操作。

可选操作：查询医嘱；确认医嘱；打印玻片；出片或取消

预期结果：医嘱列表会按照状态展示，方便医生侧统一处理后续项目。

![病理医嘱执行列表：围绕医嘱状态完成确认、打印、出片与取消。](images/m4-doctor-workflow/m4-medical-orders-overview.png)

#### 3.2 报告修订

菜单路径：诊断管理 -> 报告修订

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 进入报告修订后，先查询需要修订的病例或报告。
2. 按流程发起修订申请，并根据审批结果决定通过或驳回。

可选操作：查询病例；发起修订申请；审批通过或驳回

预期结果：修订流程应记录审批状态，便于对既有报告进行受控修正。

#### 3.3 会诊管理

菜单路径：诊断管理 -> 会诊管理

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 在会诊管理中先查询目标病例，再发起会诊并选择参与人员。
2. 随着流程推进，录入参与人意见并在满足条件后完成会诊。

可选操作：发起会诊；添加参与人；录入意见；完成会诊

预期结果：会诊页面应记录参与人、意见和完成状态，支撑多医生协作。
