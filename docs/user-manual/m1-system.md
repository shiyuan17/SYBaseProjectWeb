# M1 系统管理

系统管理覆盖用户权限、基础字典、业务模板、系统参数、编号规则与日志追踪等配置能力。

流程：系统用户 -> 角色授权 -> 基础字典 -> 业务模板与规范 -> 系统配置 -> 编号规则 -> 日志管理

### 1.人员与权限配置

用于维护系统账号、岗位和角色授权，是其他模块访问控制的基础。

#### 1.1 系统用户

菜单路径：系统管理 -> 系统用户

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 进入系统用户后，查看现有账号、岗位和状态信息。
2. 根据需要新增、编辑或启停账号，并维护岗位绑定关系。

可选操作：新增或编辑用户；启停账号；绑定岗位

预期结果：系统用户页面可维护账号生命周期和岗位归属。

![系统用户列表：维护账号、状态和岗位绑定关系。](images/m1-system/m1-users-overview.png)

#### 1.2 角色授权

菜单路径：系统管理 -> 角色授权

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 打开角色授权后，查看角色列表和对应权限码。
2. 根据岗位职责新增或调整角色，并维护菜单与功能权限。

可选操作：新增或编辑角色；分配权限码

预期结果：角色授权应能明确区分不同岗位的菜单和操作权限。

### 2.基础字典与业务字典

用于维护科室、部位、医嘱、收费和套餐等跨模块复用的基础数据。

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 进入科室字典后查看组织架构树和科室配置。
2. 按实际组织结构维护新增、停用或层级调整。

可选操作：维护组织架构；维护科室基础信息

预期结果：科室字典应支撑前台业务模块的组织选择与归属展示。

### 3.模板与规范

用于维护业务描述模板和取材规范，统一前台录入口径。

#### 3.1 描写模板

菜单路径：系统管理 -> 描写模板

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 进入描写模板后查看大体、镜下和诊断模板内容。
2. 按科室或业务类型维护模板文本，供医生与技术侧复用。

可选操作：维护大体模板；维护镜下模板；维护诊断模板

预期结果：模板维护应支持前台快速套用并保持描述口径一致。

### 4.业务配置与日志追踪

用于维护系统运行参数、编号规则和审计日志。

#### 4.1 系统配置

菜单路径：系统管理 -> 系统配置

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

操作步骤：

1. 进入系统配置后查看当前运行参数与开关项。
2. 根据部署或业务要求维护配置值，并核对变更影响。

可选操作：维护运行参数；维护系统开关

预期结果：系统配置应集中管理运行参数，避免业务页面分散维护。

菜单路径：系统管理 -> 编号规则

角色说明：建议使用管理员账号打开，便于一次性查看完整菜单、筛选条件与维护入口。

可选操作：配置病理号规则；配置标本号规则
