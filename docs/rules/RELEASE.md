# RELEASE.md — 版本发布与回滚规范

## 目标与适用范围

本文件定义项目的版本策略、环境约束、发布步骤、质量门禁与回滚要求。

- 适用对象：研发、测试、运维、发布负责人、AI 助手
- 适用范围：版本发布、热修复、环境部署、回滚处理、发布材料准备
- 规范定位：聚焦前端管理台与 Vite 构建产物的上线与回滚要求

## 强制规则

### 1. 版本策略

- 统一采用 `SemVer`：`MAJOR.MINOR.PATCH[-PRERELEASE][+BUILD]`
- `MAJOR`：不兼容路由、权限、接口契约或部署方式变更
- `MINOR`：向后兼容的新功能
- `PATCH`：缺陷修复、样式修正、低风险优化

### 2. 环境基线

| 环境      | 用途             | 部署策略       | 数据要求               |
| --------- | ---------------- | -------------- | ---------------------- |
| `dev`     | 开发自测         | 高频部署       | 可使用开发联调环境     |
| `test`    | QA 测试          | 随集成变更部署 | 可重置测试数据         |
| `staging` | UAT / 预发布验证 | 仅候选版本部署 | 对接预发布接口与配置   |
| `prod`    | 线上服务         | 仅正式版本部署 | 对接生产接口与生产配置 |

强制要求：

- 生产环境敏感配置必须来自环境变量或受控配置系统
- `staging` 必须尽量接近生产的静态资源、网关与接口拓扑
- 发布前必须确认接口地址、鉴权配置、静态资源路径、缓存策略已就绪
- 涉及浏览器能力或导出打印的版本，必须完成关键兼容验证

### 3. 标准发布流程

标准版本发布必须按以下顺序执行：

1. 从 `develop` 创建 `release/*` 分支，命名与流程遵循 `GIT_RULES.md`
2. 更新版本号、变更日志、发布说明
3. 执行构建、测试、关键页面回归、联调验证
4. 执行浏览器兼容与导出打印等专项验证并更新记录
5. 审核环境变量、接口配置、静态资源路径与缓存影响
6. 将发布分支合并到 `main` 并打 Tag
7. 同步回 `develop`
8. 触发正式部署
9. 验证登录、导航、核心页面、接口、监控与告警

### 4. 热修复流程

- 热修复必须从 `main` 创建 `hotfix/*` 分支
- 仅允许修复线上阻断问题，不得夹带需求开发
- 版本号至少递增 `PATCH`
- 合并完成后必须同步回 `develop`

### 5. 质量门禁

正式发布前必须全部满足（命令统一使用 `pnpm`，详见 `CODING_RULES.md` 的「标准验证命令」）：

- `pnpm check` 通过（循环依赖 + 依赖 + 类型 + 拼写 + 治理）
- `pnpm lint` 通过
- `pnpm test:unit` 通过；涉及关键链路时 `pnpm test:e2e` 通过
- `pnpm build` 成功
- 关键业务回归完成并有记录
- 新增或变更接口已完成联调确认
- 路由、权限、环境变量、静态资源路径已审核
- 浏览器兼容验证通过，已知问题有记录与降级说明
- 发布、回滚负责人明确，沟通通知完成

### 6. 回滚要求

满足以下任一条件时立即评估回滚：

- 登录、鉴权、核心菜单、关键页面不可用
- 请求错误率显著上升或接口配置错误导致主链路失败
- 样式资源、脚本资源、路由资源加载异常影响主要用户
- 导出、打印、图表、兼容能力出现大面积不可用

回滚必须覆盖两类内容：

- 应用回滚：恢复上一稳定版本构建产物或部署单元
- 配置回滚：恢复上一稳定的环境变量、静态资源路径或网关配置

## 推荐实践

- 在 `release/*` 阶段冻结非必要功能变更，只接受发布阻断项修复
- 提前准备发布窗口、值班人、回滚责任人与验证清单
- 对重大版本做灰度、分批或小流量验证
- 发布说明同步包含页面影响、接口影响、配置影响、兼容性影响

## 反例/禁用项

- 在发布当天临时合入未验证的新需求
- 没有 `staging` 验证就直接执行生产发布
- 把“本地能跑”当作可以免掉联调和浏览器验证的理由
- 没有配置回滚方案却修改关键环境变量或发布路径
- 热修复分支里顺带处理不相关重构

## 检查清单

- [ ] 环境变量、接口配置、静态资源路径已审核
- [ ] 浏览器兼容与关键链路验证已完成
- [ ] 发布窗口、通知、负责人、监控与告警已准备
- [ ] 回滚方案和验证步骤已确认
