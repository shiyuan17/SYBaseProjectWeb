# COMPATIBILITY_RULES.md — 浏览器与兼容性规范

## 目标与适用范围

本文件定义项目在浏览器、国产环境、导出打印与前端能力降级方面的兼容要求。

- 适用对象：前端开发者、测试人员、发布负责人、AI 助手
- 适用范围：浏览器兼容、字体图标、下载导出、图表、日期时区、能力检测与降级
- 规范定位：作为前端兼容专项规范，为开发、联调与发布提供硬性门禁

## 强制规则

### 1. 兼容基线

- 必须明确支持的浏览器范围，并在发布前完成验证记录
- 管理台默认至少覆盖现代 Chromium 内核浏览器
- 若项目要求国产环境，必须补充国产浏览器和操作系统组合验证说明
- 本地单浏览器通过不能替代目标环境兼容验证结果

### 2. 浏览器能力使用约束

- 使用 `ResizeObserver`、`IntersectionObserver`、剪贴板、文件系统、全屏、打印等能力时，必须评估兼容性与降级方案
- 新增依赖若涉及富文本、图表、导出、截图、拖拽、上传等浏览器能力，必须补充兼容性说明
- 不得默认假设字体、时区、编码、下载行为在所有环境下完全一致

### 3. 字体、图标与渲染

- 字体与图标方案必须说明来源、回退链与加载失败策略
- 表格打印、图表导出、PDF 截图等场景必须验证中文、数字、日期和特殊符号展示
- 涉及 `CSV`、`TXT`、可下载 HTML 片段或会被 Office、记事本等工具直接打开的文本内容时，必须验证目标打开方式下无乱码、无错码
- 图表颜色、字号、提示层与导出清晰度必须在目标浏览器验证

### 4. 日期、时区与格式化

- 时间展示必须明确使用的时区和格式化规则
- 日期选择、表格时间列、导出文件名中的时间字段必须保持一致
- 禁止依赖运行环境的默认本地化行为而不做显式控制

### 5. 下载、上传、导出与打印

- 下载必须处理文件名编码、跨浏览器触发方式和失败提示
- `CSV`、`TXT` 等文本导出必须明确编码方案，默认使用 `UTF-8`；若因外部系统或办公软件兼容性必须使用其他编码，必须记录原因、适用范围和验证结论
- 上传必须校验文件大小、格式、浏览器限制和异常中断
- 打印、截图、导出能力必须定义不支持时的兜底提示或替代路径

### 6. 发布门禁

- 新功能涉及关键浏览器能力时，必须补充验证范围和结论
- 存在已知不兼容项时，必须记录影响范围、降级方式、责任人和整改计划
- 无兼容验证、无降级说明或无记录的问题，不得宣称具备发布状态

## 推荐实践

- 维护浏览器兼容矩阵，记录浏览器、版本、系统、问题结论和处理状态
- 对关键能力采用能力检测而不是 UA 猜测
- 为导出、打印、图表、自适应和大屏场景准备专项回归清单
- 让产品、设计、测试提前知道兼容边界，而不是在上线前集中暴露

## 反例/禁用项

- 只在单一 Chrome 版本验证就宣称“浏览器兼容”
- 使用新浏览器能力却没有降级策略或用户提示
- 通过手工修改线上配置规避兼容问题，却不补文档和验证记录
- 把导出乱码、打印样式错乱、图表白屏当作“个别机器问题”忽略

## 检查清单

- [ ] 已明确支持的浏览器与目标环境范围
- [ ] 涉及浏览器能力的功能已评估兼容性与降级方案
- [ ] 字体、图标、导出、打印、图表展示已完成关键验证
- [ ] 日期、时区、编码与文件名策略一致
- [ ] 中文、特殊符号、文件名、`CSV/TXT` 文本导出在目标浏览器和常见打开方式下无乱码
- [ ] 已知不兼容项已记录影响、兜底与整改计划
