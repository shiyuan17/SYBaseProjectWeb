# VUE_TS_RULES.md — Vue 3 与 TypeScript 规范

## 目标与适用范围

本文件定义 `Vue 3 + Vite + TypeScript` 技术栈下的组件、组合式函数与类型实现规则。

- 适用对象：前端开发者、组件维护者、AI 助手
- 适用范围：`.vue` 单文件组件、组合式函数、TypeScript 类型模块
- 规范定位：统一组件写法、类型表达方式与异步逻辑组织方式

## 强制规则

### 1. 基础写法

- 统一使用 `script setup + TypeScript`
- 禁止在新代码中回退到 Options API，除非需要兼容既有存量模块
- 组件必须显式定义 `props`、`emits`、插槽契约或暴露能力

### 2. 组件职责边界

- 页面组件负责组装，不负责承载大量可复用细节
- 共享组件必须保持业务无关，不直接依赖模块内部 store 或接口模型
- 超过单一职责的组件必须拆分，避免形成超大 SFC
- 禁止把接口请求、复杂图表配置、格式化工具、权限判断全部堆在同一个 `.vue` 文件

### 3. TypeScript 约束

- 所有公开能力必须有明确类型定义
- 类型命名必须表达语义，例如 `UserListQuery`、`UserTableRow`
- 避免滥用 `any`、`as unknown as`、宽泛断言来掩盖类型问题
- DTO、ViewModel、组件 Props 类型应分层定义，不混用

### 4. 组合式函数

- 组合式函数统一使用 `useXxx` 命名
- 组合式函数负责封装可复用状态和行为，不负责模块初始化顺序以外的全局副作用
- 异步组合式函数必须提供加载、错误、重试或刷新能力
- 组合式函数返回值必须稳定，避免把过多临时细节暴露给调用方

### 5. 模板与响应式使用

- 模板中避免堆积复杂表达式、链式访问和重复格式化逻辑
- 响应式状态应按用途划分，不要把整块接口响应对象直接变成页面主状态
- `computed`、`watch`、`watchEffect` 必须有明确用途，禁止为了“能跑”随意堆监听
- 需要清理的副作用、定时器、事件监听、图表实例必须在组件销毁时释放

## 推荐实践

- 页面拆分为 `View + Section + Shared Component` 层级，控制单文件复杂度
- 对模块内类型进行集中导出，便于服务层、store 和页面共享
- 为表格筛选、图表数据转换、表单提交流程提炼组合式函数
- 通过 `defineProps`、`defineEmits`、`defineExpose` 明确组件边界

## 反例/禁用项

- 单个页面组件超过数百行且同时承担请求、转换、图表、弹窗和权限逻辑
- 使用 `any` 覆盖接口模型、表单值、路由参数等关键类型
- 在模板里直接调用复杂函数或包含多层空值分支
- 通过全局变量或隐式副作用让组件工作

## 检查清单

- [ ] 使用 `script setup + TypeScript`
- [ ] Props、Emits、公开类型定义清晰
- [ ] 组件职责单一，无明显超大 SFC
- [ ] 组合式函数命名与边界清晰
- [ ] 响应式副作用、监听与销毁逻辑完整
